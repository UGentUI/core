### Color System

Our color system is built on UGent's distinctive brand colors, carefully expanded into functional palettes. Tints and shades follow a consistent numerical scale from 50 to 950 in increments of 100, with step 600 serving as the primary color in each palette. To ensure consistency and flexibility across applications, these colors are defined as **design tokens**. The tokens are stored in a [repository](https://github.com/ugent/design-tokens) as a Single Source of Truth and are available in different formats.

Saturated colors enhance user experience by providing meaningful visual cues, highlighting important UI elements, and creating consistent patterns across our applications.

### Primary and Semantic Colors

The **primary color** is used for main UI elements, brand representation, and primary actions. **Semantic colors**—such as success, warning, and danger—provide clear visual feedback for specific states. All of these colors use 600 as their base shade.

### Usage

To apply the default styles globally across your project, import the UGentUI style sheet:

```html
<link rel="stylesheet" href="@ugent-ui/core/style.css" />
```

Design color tokens are represented as CSS variables, which are automatically applied to the root element (`:root`). This ensures consistent styling throughout your application while allowing flexibility for customization. Additionally, **UGent UI components inherit these styles**, ensuring they remain visually consistent with the design system and automatically adapt to theme changes, including dark mode.

All CSS color variables are available globally, allowing you to use them in your custom styles:

```css
.custom-element {
  background-color: var(--ug-color-neutral-100);
  color: var(--ug-color-primary-600);
}
```

For a full list of available CSS color variables, refer to the [CSS Color Variables Reference](#available-css-color-variables).

### Accessibility

The UGentUI color system is designed to ensure readability and usability for all users, including those with visual impairments. We follow WCAG 2.1 guidelines to maintain accessible contrast ratios, ensuring that text and UI elements remain distinguishable in various lighting conditions and for different levels of vision.

To meet WCAG 2.1 AA accessibility standards:

- **Text and interactive elements** must have a contrast ratio of **at least 4.5:1** against their background.
- **Large text** (18px+ or 14px bold) requires a minimum contrast ratio of **3:1**.
- **Non-text elements** (icons, buttons, borders) should have a contrast ratio of **at least 3:1** against adjacent colors.

To ensure compliance with accessibility standards, UGentUI integrates automated contrast testing directly into Storybook. Each component is evaluated using Storybook's accessibility add-on, which checks for contrast ratio issues and WCAG compliance.

When using CSS color variables in your custom styles, it's essential to verify that your color combinations meet WCAG accessibility standards. While UGentUI's default color combinations are tested for accessibility, **custom combinations should be independently validated**.

### Dark Mode Support

Our color system includes built-in support for dark mode through the `data-color-mode` attribute. Colors are automatically inverted and adjusted for optimal contrast and readability in dark mode. To ensure consistency, always use primary and semantic tokens, as they adapt seamlessly between light and dark modes.

To enable dark mode, you can toggle the `data-color-mode` attribute to `"dark"` or `"light"` (default) on the root element of your application.

```html
<html lang="en" data-color-mode="dark">
  <head>
    <link rel="stylesheet" href="@ugent-ui/core/style.css" />
  </head>
  <body>
    <!-- Colors are automatically inverted and adjusted for dark mode -->
    <div style="background-color: var(--ug-color-neutral-100);">
      <h1 style="color: var(--ug-color-neutral-950);">Dark Mode</h1>
    </div>
  </body>
</html>
```
